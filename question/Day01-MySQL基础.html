前言

SQL是 结构化查询语言（Structure Query Language） ：专门用来与数据库通信的语言

SQL的优点：

  • 不是某个特定数据库供应商专有的语言，几乎所有的DBMS都支持SQL
  • 简单易学
  • 虽然简单，但实际上是一种强有力的语言，灵活使用其语言元素，可以进行非常复杂和高级的数据库操作

一、数据库的好处

  • 持久化数据到本地
  • 可以实现结构化查询，方便管理

二、数据库的概念

2.1 DB

数据库（database） ：存储数据的“仓库”。它保存了一系列有组织的数据。

2.2 DBMS

数据库管理系统（Database Management System） 。数据库是通过DBMS创建和操作的容器。

常用的数据库管理系统：MySQL、Oracle、DB2、Sqlserver等

2.3 程序员、数据库与管理系统之间的关系

程序员使用SQL语言通过DBMS对数据库进行相关操作（增删改查）

三、数据库的特点

  • 将数据放到表中，表再放到数据库中

  • 一个数据库可以有多个表，每个表都有一个名字，用来标识自己。表明具有唯一性
  • 表具有一些特性，这些特性定义了数据在表中如何存储，类似于Java“类”的设计
  • 表由列组成，我们也称为字段。所有表都是由一个或多个列组成的，每一列类似Java中的“属性”
  • 表中的数据是按行存储的，每一行类似于Java中的“对象”

四、MySQL的安装卸载

4.1 安装

双击安装包， 点击Next ， 选择Custom 自定义安装， 点击Next

点击 Install ，等待安装完成

点击Finish，环境安装完成

进入配置路径，如果配置路径界面不小心被关闭了，可以通过安装目录下路径：MySQL->MySQL Server 5.5->bin再次打开（ 一般这一步可以跳过 ）

进入配置界面，点击 Next ，选择默认的精确配置即可

选择开发机即可

选择 多功能型 ， Next ，然后再 Next

选择同一时刻允许的并发连接数，一般选择 第一个（20） 即可

端口号选择 默认的3306 即可（如果是在实际开发中是需要修改的）

选择utf8

Name可以修改（一般不修改）

设置密码（一般简单点）

直接执行即可，等待四个选项完成

安装完毕，可以使用了

设置MySQL自启动或手动启动

计算机管理方式启动 -> 开始 -> 右击“计算机” -> 选择“管理” -> 服务

右击MySQL可修改状态，双击可设置手动启动

4.2 卸载MySQL

  • 先在控制面板->程序->程序和功能卸载程序
  • 在自己MySQL的安装路径将安装的文件夹删除
  • 在C盘的ProgramData（隐藏文件）中将MySQL文件夹删除

一般经过以上步骤就卸载完成了，如果再次安装还有问题，按照以下步骤操作

清理注册表：进行cmd界面，输入regedit，安装上面清理注册表的步骤清理即可

注意：如果安装不成功，或者有问题，卸载重装就行
